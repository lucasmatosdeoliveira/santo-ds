## How this Storybook is organised

- **Foundations** — the design language. Color roles, type scale, motion
  curves, spacing, radius, breakpoints, density, z-index, state overlays,
  data-viz palettes, and the rationale behind each token layer.
- **Components** — the React building blocks, grouped by intent:
  - **Actions** — Button, IconButton, SocialButton, Toggle, Chip
  - **Inputs** — Field, Input, Textarea, PasswordField, DateField,
    Checkbox, Switch, RadioGroup, Select, Combobox, Slider
  - **Display** — Text, Avatar, Badge, Card, Separator, Skeleton, Progress
  - **Feedback** — Alert
  - **Overlay** — Dialog, Popover, Tooltip, DropdownMenu
  - **Navigation** — Tabs, Accordion

Every component page has a **Playground** story with full controls, plus
showcase stories for the most common use-cases.

## Architecture

```
@santo-ds/tokens   →  Figma variables → Style Dictionary → CSS / JS / TS / Tailwind preset
@santo-ds/ui       →  React components (Radix primitives + class-variance-authority)
@santo-ds/docs     →  This Storybook
```

Tokens flow in a single direction: **Figma → JSON → CSS variables →
Tailwind utilities → React components**. Every component variant resolves
to a token, so a single token change propagates to every consumer.

## Quick start

```bash
pnpm install
pnpm --filter @santo-ds/tokens build
pnpm --filter @santo-ds/docs storybook   # opens on http://localhost:6006
```

```tsx
import { Button, Card, CardContent, Text } from "@santo-ds/ui";
import "@santo-ds/tokens/css";

export function Welcome() {
  return (
    <Card className="w-80">
      <CardContent className="flex flex-col gap-3 p-6">
        <Text variant="title-lg">Welcome to SantoDS</Text>
        <Text variant="paragraph-md">
          Tokens, components and theming — already wired together.
        </Text>
        <Button>Get started</Button>
      </CardContent>
    </Card>
  );
}
```

## Theming

Toggle the theme with the **paint-roller icon** in the toolbar above —
it switches the `data-theme` attribute on the preview iframe. In your
own app, set `data-theme="dark"` on `<html>` (or any ancestor) and the
tokens swap automatically.

```html
<html data-theme="dark">
  <!-- everything below resolves to dark-mode tokens -->
</html>
```

## Accessibility

The **a11y** panel runs `axe-core` on every story against
`wcag2a / wcag2aa / wcag21a / wcag21aa`. Foundation contrast pairs are
validated in CI by `pnpm --filter @santo-ds/tokens validate:contrast` —
the build fails if any role pair drops below WCAG AA.

## Conventions

- All components are **forward-ref** and accept `className`. Tailwind
  utilities and the `cn()` helper compose cleanly with the variants.
- All variants are powered by **class-variance-authority** (`cva()`),
  so `<Button variant="primary" size="lg" />` types match the runtime.
- Sizing reads **component tokens** (`h-btn-md`, `size-icon-btn-lg`,
  `rounded-input-sm`), not raw spacing scales — change the token, every
  consumer updates.
- Composite typography is applied via the auto-generated
  `.sds-text-{variant}` utility classes (driven by the `Text`
  component).

## Token update flow

1. Edit Figma variables in **Santo DS — Foundation**.
2. Export via the **Design Tokens** plugin → save as
   `packages/tokens/design-tokens.json`.
3. Run `pnpm tokens:build` — Style Dictionary regenerates CSS, JS, TS
   and the Tailwind preset.
4. Commit. Consumers (`@santo-ds/ui`, `@santo-ds/docs`) pick the change
   up via the pnpm workspace.

## Naming

Every CSS variable follows `--sds-<category>-<subcategory>-<variant>`:

- `--sds-color-primitive-neutral-500`
- `--sds-color-semantic-background-default-soft`
- `--sds-size-16`
- `--sds-border-radius-8`
- `--sds-effect-shadow-md`
- `--sds-typography-headline-lg-font-size`
